# ADR-004: Monetization Path

## Status

Proposed

## Context

The CiviCore Resource and Template Library is intended to become a long-term public trust and traffic asset for CiviCore IT Solutions.

The project should first publish useful resources and build organic discovery before adding monetization layers.

Monetization must not reduce content quality, confuse users, or create premature complexity before the library has enough useful public content.

## Decision

The proposed monetization path is staged:

1. Free traffic first.
2. AdSense later.
3. Premium template packs later.
4. CiviCore service lead capture later.
5. SheetBot contextual links later.

No ads, payment flows, premium packs, or lead capture systems should be added until the content structure, review policy, disclaimers, and basic SEO foundation are ready.

## Consequences

Prioritizing free traffic first helps the project build topical authority, public usefulness, and search visibility before monetization.

AdSense should wait until the site has sufficient original content and a reasonable user experience.

Premium packs can be introduced later once free templates reveal demand and common use cases.

CiviCore service lead capture should be soft and contextual, not aggressive.

SheetBot links should be added only where they are relevant to productivity, spreadsheet automation, billing, reporting, document workflows, or office digitalization topics.

## Alternatives Considered

### Monetize Immediately

Rejected for the initial phase because ads, payments, and lead capture can distract from building useful public content and may create unnecessary implementation work.

### Free-Only Permanently

Simple and trust-building, but it may underuse the library's long-term potential as a CiviCore traffic, service, and product asset.

### Staged Monetization

Recommended because it keeps the initial project simple while preserving future paths for AdSense, premium templates, service inquiries, and contextual SheetBot adoption.
